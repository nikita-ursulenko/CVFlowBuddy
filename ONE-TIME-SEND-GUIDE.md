# 📤 Гайд по разовой отправке CV

## 🎯 Описание

Разовая отправка CV - это функция отправки резюме на заданное количество вакансий **за один запуск**, с автоматическим мониторингом прогресса и защитой от зависания.

## ✨ Возможности

### 1. **Блокировка кнопки**
- ✅ Кнопка "Разовая отправка" блокируется на время выполнения
- ✅ Показывается статус "Отправка..."
- ✅ Невозможно нажать повторно до завершения

### 2. **Мониторинг прогресса**
- ✅ Отображение текущего прогресса: "Отправка CV: 5 / 20"
- ✅ Автоматическая проверка каждые 15 секунд
- ✅ Анимированный прогресс-бар с индикатором активности
- ✅ Обновление в реальном времени

### 3. **Проверка состояния агента**
- ✅ Отслеживание активности браузера
- ✅ Проверка наличия свежих отправок (за последние 30 секунд)
- ✅ Автоматическое обнаружение зависания

### 4. **Автоматический перезапуск**
- ✅ Если нет прогресса 45 секунд (3 проверки × 15 сек) → автоперезапуск
- ✅ Уведомление пользователя о перезапуске
- ✅ Продолжение отправки после перезапуска
- ✅ Агент закрывается и запускается заново

### 5. **Работа до конца**
- ✅ Отправка продолжается до достижения заданного количества
- ✅ Автоматическая обработка ошибок
- ✅ Финальный отчет с результатами

## 📋 Как использовать

### Шаг 1: Подготовка
```
1. Загрузите CV файл (раздел "CV")
2. Проанализируйте CV с помощью AI (раздел "AI")
3. Войдите в аккаунт Lucru.md (раздел "Агент" → "Войти в аккаунт")
```

### Шаг 2: Настройка количества
```
1. Откройте раздел "Агент"
2. Нажмите на иконку настроек ⚙️
3. Установите "Максимум CV в день" (например, 20)
4. Сохраните настройки
```

### Шаг 3: Запуск
```
1. Нажмите кнопку "Разовая отправка" ⚡
2. Дождитесь завершения (кнопка заблокирована)
3. Следите за прогрессом в реальном времени
```

## 🛡️ Защита от зависания

### Как работает

```
Каждые 5 секунд:
├─ Проверка прогресса отправки
├─ Проверка активности браузера
├─ Проверка свежих отправок (< 30 сек)
└─ Обновление UI (Отправка CV: X / 100)

Если НЕТ прогресса 24 раза подряд (2 минуты):
├─ 🔄 Закрытие старой сессии агента
├─ ⏸️  Пауза 2 секунды
├─ 🚀 Автоматический перезапуск
└─ ▶️  Продолжение отправки
```

### Уведомления

```
⚠️ Агент завис (2 мин без прогресса). Выполняется перезапуск...
🔄 Агент перезапущен. Продолжаем отправку...
```

## ⚙️ Параметры

### Настройки агента

| Параметр | Описание | По умолчанию |
|----------|----------|--------------|
| `maxCVDaily` | Количество вакансий за запуск | 20 |
| `intervalHours` | Интервал между запусками (для авто) | 4 часа |
| `headless` | Скрытый режим браузера | true |
| `minMatchScore` | Минимальное соответствие вакансии | 70% |

### Таймауты

| Таймер | Значение | Назначение |
|--------|----------|------------|
| Progress Check | 5 сек | Проверка прогресса и обновление UI |
| Recent Activity | 30 сек | Определение свежести отправок |
| No Progress Limit | 2 мин | Лимит перед перезапуском (24×5 сек) |
| Request Timeout | 10 мин | Максимальное время запроса |
| Modal Wait | 5 сек | Ожидание загрузки модального окна |

## 🔧 Troubleshooting

### Проблема: Кнопка не разблокируется

**Решение:**
```javascript
// Откройте консоль браузера (F12)
localStorage.removeItem('agent_auto_apply_progress');
location.reload();
```

### Проблема: Зависание без перезапуска

**Проверьте:**
1. Сервер агента работает: `http://localhost:5050/api/health`
2. Есть ли sessionId: Проверьте в разделе "Агент" → "Конфигурация"
3. Браузер агента открыт: Должно быть окно Playwright Chromium

**Ручной перезапуск:**
```bash
# В терминале
pkill -f 'agent-server-simple'
cd /home/parallels/GitHub/cvflow-buddy
node agent-server-simple.js
```

### Проблема: Сессия истекла

**Решение:**
1. Нажмите "Выйти" в разделе "Агент"
2. Войдите заново через "Войти в аккаунт"
3. Повторите разовую отправку

### Проблема: Нет прогресса вообще

**Проверьте:**
```javascript
// 1. Статус сервера
fetch('http://localhost:5050/api/health')
  .then(r => r.json())
  .then(d => console.log('✅ Сервер:', d));

// 2. Статистика отправок
fetch('http://localhost:5050/api/stats')
  .then(r => r.json())
  .then(d => console.log('📊 Статистика:', d));

// 3. Статус агента
const sessionId = localStorage.getItem('cvflow_session_id');
fetch('http://localhost:5050/api/agent/status', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ sessionId })
})
  .then(r => r.json())
  .then(d => console.log('🤖 Агент:', d));
```

## 💡 Советы и рекомендации

### ✅ Лучшие практики

1. **Начинайте с малого**
   - Первый раз: 5-10 вакансий
   - Убедитесь что всё работает
   - Потом увеличивайте до 20-30

2. **Следите за прогрессом**
   - Держите вкладку открытой
   - Наблюдайте за обновлениями
   - Проверяйте логи в консоли (F12)

3. **Не закрывайте вкладку**
   - Мониторинг работает в JavaScript
   - Закрытие вкладки = потеря контроля
   - Используйте фоновый режим агента вместо разовой отправки

4. **Проверяйте статистику**
   - После каждой отправки
   - В разделе "Dashboard" → "Статистика"
   - Убедитесь что CV реально отправляются

### ⚠️ Частые ошибки

1. **Запуск без авторизации**
   ```
   ❌ Сначала войдите в аккаунт
   ✅ Раздел "Агент" → "Войти в аккаунт"
   ```

2. **Запуск без AI анализа**
   ```
   ❌ Сначала загрузите и проанализируйте CV
   ✅ Раздел "AI" → "Анализировать CV"
   ```

3. **Повторное нажатие кнопки**
   ```
   ❌ Кнопка заблокирована = отправка идёт
   ✅ Дождитесь завершения
   ```
