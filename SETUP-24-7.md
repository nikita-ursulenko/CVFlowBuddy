# 🚀 Настройка CVFlow Buddy для работы 24/7

## 1️⃣ Отключение спящего режима

Чтобы Ubuntu не уходил в спящий режим и сервер работал 24/7:

```bash
cd /home/parallels/GitHub/cvflow-buddy
sudo bash disable-sleep.sh
```

После этого:
- ✅ Система не будет засыпать
- ✅ Экран не будет отключаться
- ✅ Серверы будут работать постоянно

---

## 2️⃣ Автоматический запуск серверов при загрузке

Создайте systemd сервис для автозапуска:

```bash
sudo nano /etc/systemd/system/cvflow-buddy.service
```

Вставьте:

```ini
[Unit]
Description=CVFlow Buddy Servers
After=network.target

[Service]
Type=simple
User=parallels
WorkingDirectory=/home/parallels/GitHub/cvflow-buddy
ExecStart=/usr/bin/bash /home/parallels/GitHub/cvflow-buddy/START-SERVERS.sh
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
```

Сохраните (Ctrl+O, Enter, Ctrl+X) и запустите:

```bash
# Перезагрузить конфигурацию systemd
sudo systemctl daemon-reload

# Включить автозапуск
sudo systemctl enable cvflow-buddy

# Запустить сервис
sudo systemctl start cvflow-buddy

# Проверить статус
sudo systemctl status cvflow-buddy
```

## 3️⃣ Проверка автоотправки каждый час

Автоотправка настроена и работает так:

### Как работает:

1. **Открываете страницу** `http://10.211.55.5:5004/sites`
2. **Нажимаете "Запустить"** - агент стартует
3. **Первая отправка** - сразу (через 2 секунды)
4. **Следующие отправки** - каждый час автоматически

### Логика:

```
Запуск → Первая отправка → Закрытие браузера → Ожидание 1 час → Повторная отправка → ...
```

### Проверка времени следующего запуска:

Откройте консоль браузера (F12) и найдите:

```
⏱️ Задача выполнялась 0.5ч назад. Следующий запуск через 0.5ч (интервал: 1ч)
```

## 5️⃣ Настройки автоагента

В интерфейсе можно изменить:

- **Интервал**: 1ч, 2ч, 3ч, 6ч, 12ч, 24ч
- **Количество CV**: 5, 10, 20, 50, 100 за раз
- **Режим**: Headless (фоновый) или с браузером

Настройки сохраняются в localStorage и применяются при следующем запуске.
