# 🔧 Решение проблем

## 🔄 Агент "включается-выключается"

### Причины:

1. **Сессия истекла** - cookies стали невалидными
2. **Браузер закрылся** - activeAgent потерял соединение
3. **Ошибка авторизации** - неверные credentials
4. **Таймаут** - операция заняла больше времени чем ожидалось

### Решение:

#### 1. Проверьте cookies

```bash
ls -la /home/parallels/GitHub/cvflow-buddy/lucru-cookies.json
cat /home/parallels/GitHub/cvflow-buddy/lucru-cookies.json | head -20
```

Если файл старый (больше 24 часов) - пере логиньтесь:

1. Остановите агента
2. Нажмите "Войти в Lucru.md"
3. Введите email/пароль
4. Дождитесь сообщения "✅ Успешная авторизация"
5. Запустите агента снова

#### 2. Проверьте логи планировщика

Откройте консоль браузера (F12) и найдите:

```
✨ Статус изменился на running, автоматически запускаем планировщик
🚀 Планировщик: Запускаем первую задачу через 2 секунды...
🚀 Планировщик: Выполняем первую задачу...
```

Если видите:
```
❌ Не можем запустить планировщик: статус = stopped
```

Значит агент не запущен. Нажмите кнопку "▶️ Запустить".

#### 3. Проверьте sessionId

В консоли браузера:

```javascript
localStorage.getItem('cvflow_lucru_session')
// Должна быть строка типа: "session_1759916..."
```

Если `null` - нужно авторизоваться заново.

#### 4. Проверьте backend логи

В терминале где запущен `agent-server-simple.js` ищите:

```
✅ Новая сессия создана с cookies
✅ Авторизация через cookies успешна!
```

Или ошибки:
```
❌ Ошибка авторизации: ...
⚠️ Cookies невалидны или истекли
```

### Быстрое решение:

```bash
# 1. Остановите агента в UI
# 2. Удалите старые cookies
rm /home/parallels/GitHub/cvflow-buddy/lucru-cookies.json

# 3. Перезапустите backend
pkill -f "agent-server-simple"
cd /home/parallels/GitHub/cvflow-buddy && node agent-server-simple.js &

# 4. В UI: Войти → Ввести credentials → Запустить агента
```

## ⏱️ Процесс останавливается после пары вакансий

### Причины:

1. **Таймаут слишком короткий** - исправлено (теперь 10 минут)
2. **Браузер закрылся** - исправлено (теперь 30 секунд)
3. **Элементы не найдены** - вакансии без кнопки CV пропускаются
4. **Модальное окно не закрылось** - зависло

### Решение:

#### 1. Проверьте логи backend

```
📊 ПРОГРЕСС: 3 / 20 вакансий
   ✅ Отправлено: 2 | ⏭️ Пропущено: 1
```

Если видите что процесс остановился на конкретной вакансии:

```
📄 5. Senior Developer
   🔗 /vacancy/12345
   ⚠️ Кнопка CV не найдена - копируем ссылку и скрываем
```

Это нормально - вакансия без кнопки CV пропускается автоматически.

#### 2. Проверьте headless режим

Откройте настройки агента:
- Если "Фоновый режим" выключен - браузер будет видимым
- Можно увидеть что именно делает агент
- Полезно для отладки

#### 3. Увеличьте лог уровень

В `agent-server-simple.js` уже добавлено детальное логирование:

```javascript
console.log(`📊 ПРОГРЕСС: ${i + 1} / ${maxJobs} вакансий`);
console.log(`   ✅ Отправлено: ${appliedCount}`);
console.log(`   ⏭️ Пропущено: ${skippedCount}`);
```

Следите за этим в терминале backend.

## 💾 Настройки не сохраняются

### Проверка:

```javascript
// В консоли браузера (F12):
localStorage.getItem('cvflow_lucru_config')
```

### Причины:

1. localStorage заблокирован
2. Приватный режим браузера
3. Cookies отключены

### Решение:

```javascript
// Принудительное сохранение:
localStorage.setItem('cvflow_lucru_config', JSON.stringify({
  site: 'lucru.md',
  name: 'Lucru.md Agent',
  settings: {
    maxCVDaily: 20,
    intervalHours: 3,
    headless: true
  }
}))

// Проверка:
JSON.parse(localStorage.getItem('cvflow_lucru_config')).settings
```
